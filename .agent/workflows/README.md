# Workflows — index

Workflows are compact operating procedures for common task types.

## Available workflows

- `audit.md`
- `bugfix.md`
- `command-add.md`
- `db-change.md`
- `docs-update.md`
- `feature.md`
- `module-create.md`

## Usage rule

- a workflow does not replace planning for non-trivial work,
- start with `change-classifier`,
- if the task is larger than a small fix, pair the workflow with `planning-standard`,
- after execution, synchronize the workflow result with project memory and the closeout report.
