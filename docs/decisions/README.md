# Decisions — guidance

Use this directory for accepted architectural or operational decisions.

## A decision record should include

- title,
- context,
- decision,
- consequences,
- status.

## Do not put here

- raw audits,
- temporary notes,
- backlog ideas,
- issue dumps.

Those belong elsewhere:
- audits → `docs/`
- backlog → `ROADMAP.md`
- unresolved defects → `docs/CODE_ISSUES_TO_FIX.md`
